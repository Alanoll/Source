Installation project via Grunt.js

Installation

To install project you need to install grunt. For this you need to run commands below:

$ cd Source $ npm install $ grunt init

After that all necessary files will be copied in a folder named Spec, next to Source folder. If you want to change target location, you need to change value of pathToSpecs property in a options file gruntOptions.js. You can use both relative and absolute pathes. Relative path should depend on Gruntfile.js.

Update project

After updating to the new version, you may need to deploy new Source.js to your project. For this you should run :

$ grunt update

All necessary Source.js files will be minimized, uglified and copied to your project folder.

Also you can run grunt runWatch,and if some file has changed, Source.js will be deployed locally. This mode is needed only for developing Source.js. Usually, you don't need to run watch task.

Deploy to the server via sftp

For deploying to the server, you need to fill server access options in file gruntOptions.js according to your options. Also you need to change name/password in .ftppass. After these changes run:

$ grunt deployServer

All necessary Source.js files will be minimized, uglified and uploaded to your server.
